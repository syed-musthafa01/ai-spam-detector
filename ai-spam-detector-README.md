# AI Spam Detector

A machine learning web app that classifies text messages as **Spam** or **Not Spam**, with a Python/FastAPI backend, a scikit-learn model, and a minimal HTML/JS frontend. Built with the help of AI coding tools (AI IDEs / AI models).

**Live demo** https://ai-spams-detector.netlify.app/

---

## How it works

```
[ dataset/spam.csv ]
        │
        ▼  (offline, one-time)
[ model/train_model.py ]  ──▶  spam_model.pkl + vectorizer.pkl
        │
        ▼  (loaded at server startup)
[ api/main.py — FastAPI ]  ◀──POST /predict── [ frontend/index.html ]
        │
        ▼ (optional)
[ Postgres: predictions table ]
```

1. **Training (offline, run once):** `model/train_model.py` loads the classic SMS Spam Collection dataset (`dataset/spam.csv`), converts each message into a TF-IDF vector, and trains a Multinomial Naive Bayes classifier. The trained model and vectorizer are pickled to `model/spam_model.pkl` and `model/vectorizer.pkl`.
2. **Serving:** `api/main.py` loads both pickle files into memory on startup and exposes a single endpoint, `POST /predict`, which accepts `{ "text": "..." }` and returns a prediction with a confidence score.
3. **Logging (optional):** every prediction is inserted into a Postgres `predictions` table if database credentials are configured. If the DB is unreachable, the API still responds — it just skips logging.
4. **Frontend:** a single static HTML page with a textarea and button, calling the API directly via `fetch`.

## Setup & running locally

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. (Optional) Retrain the model
The repo already ships pre-trained `spam_model.pkl` and `vectorizer.pkl`, so this step is optional. To retrain:
```bash
cd model
python train_model.py
```
> Note: this script uses a relative path (`../dataset/spam.csv`), so it must be run from inside the `model/` directory.

### 3. (Optional) Configure the database
Set these environment variables if you want predictions logged:
```bash
DB_HOST=...
DB_NAME=...
DB_USER=...
DB_PASSWORD=...
DB_PORT=6543   # defaults to 6543 if not set
```
The target table must already exist:
```sql
CREATE TABLE public.predictions (
    id SERIAL PRIMARY KEY,
    input_text TEXT,
    prediction TEXT,
    confidence FLOAT
);
```
If no DB is configured, the API works fine and simply skips logging.

### 4. Run the API
```bash
cd api
uvicorn main:app --reload
```
API will be available at `http://127.0.0.1:8000`.

### 5. Open the frontend
Open `frontend/index.html` in a browser. By default it's pointed at the deployed Render API — update the URL inside the `<script>` tag if you want it to talk to your local server instead.

## API reference

### `POST /predict`

**Request body:**
```json
{ "text": "Win a free iPhone now!" }
```

**Response:**
```json
{
  "input_text": "Win a free iPhone now!",
  "prediction": "Spam",
  "spam_confidence": 91.42
}
```

Classification threshold: messages with spam probability > 50% are labeled `"Spam"`.

## Known limitations / next steps

- **No DB migration script** — the `predictions` table must be created manually; it isn't generated by any code in this repo.
- **CORS is fully open** (`allow_origins=["*"]`) — fine for a demo, should be restricted before any real production use.
- **No retraining pipeline** — the model is static; new spam patterns require manually rerunning `train_model.py` and redeploying.
- **No error handling in the frontend** — if the API is unreachable or asleep (Render free tier sleeps on inactivity), the page gives no feedback.
- **Fixed decision threshold (0.5)** — not currently tunable without editing code.
